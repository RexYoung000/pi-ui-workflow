# UI 工作流 —— 仅对齐需求

只跑**阶段 1：对齐需求**。适合用户想先理清自己想做什么、还不想花 token 跑分析师的场景。

## 执行步骤

**重要前置检查**：如果 $@ 为空（用户只输入了 `/ui-align` 没有带任何描述），**先不要进入需求对齐**。立即用 `ask_user_question` 问用户：

> **「你想设计什么产品？请简单描述一下。」**

拿到用户描述后再继续。

1. **扫项目**：用 `read` / `ls` / `find` 检查 `README.md`、`AGENTS.md`、`docs/`、`package.json`、已有 UI 代码等
2. **如果用户在调用时给了描述**（`$@`），把这段描述作为主要输入
3. **用 `ask_user_question` 问 2-3 个关键问题**（每问最多 4 个问题，每个 2-4 个选项）。常见关键决策：
   - 产品形态（Web App / 移动 / 桌面 / 多端）
   - 目标用户（专业用户 / 大众 / 企业内部）
   - 风格调性（专业克制 / 活泼现代 / 极简优雅 / 奢华精致）
   - MVP 范围（先做什么、后做什么）
4. **拿到回答后**，用一段中文总结「我理解的需求」让用户最后确认
5. **产出一份简洁的需求摘要**写到 `docs/ui-need-summary.md`，包含：
   - 产品定位
   - 目标用户
   - 核心功能（按优先级）
   - 关键页面
   - 约束与偏好
   - 待确认问题（如果有）

## 重要

- **不要调用 `ui_workflow_subagent` 工具**，这一步你自己做
- **不要做形态/视觉决策**，那是后续阶段的事
- 卡点就停下问用户，不要脑补关键需求

## 完成后

告诉用户：
- 摘要文件保存在哪里
- 如果想继续走完整流程，可以用 `/ui-review` 或 `/ui-workflow`
